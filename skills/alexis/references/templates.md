# 输出模板

## 需求简报（alexis-new）

```markdown
# 需求简报

## 1) 目标
...

## 2) 范围
- 包含：...
- 不包含：...

## 3) 约束
...

## 4) 优先级
...

## 5) 交付物
...

## 6) 验收标准
...

## 未决事项
- ...（若无写“无”）
```

## 执行方案（alexis-plan）

```markdown
# 执行方案

## 目标与背景
...

## 执行步骤
1. ...
2. ...

## 里程碑与时间
- M1：...

## 风险与应对
- 风险：...
  - 应对：...

## Task List
### Task-01: <任务名称>
- 输入（当前现状）：...
- 输出（完成产物）：...
- 验收条件：...
- 执行说明（最小上下文）：文件/命令/依赖/风险...
```

## plan_review 修订回应记录

```markdown
## 修订回应记录（YYYY-MM-DD HH:mm）

1. 评审意见：...
   - 处理结论：采纳/不采纳
   - 处理说明：...
   - 对应方案变更：...
```

## 验收失败改进映射（写入 design.md）

```markdown
## 验收失败改进映射（YYYY-MM-DD HH:mm）

1. 验收意见：...
   - 影响：...
   - 计划改进：...
   - 对应 Task：Task-03
```

## 验收报告（alexis-accept）

```markdown
# 验收报告

## 验收范围
...

## 前置条件核查
- Task 完成情况：全部完成/未全部完成
- 未完成 Task：...（若无写“无”）

## 标准逐条核对
1. 标准：...
   - 结果：通过/不通过
   - 证据：...

## 总体结论
- 结论：通过/不通过

## 改进建议
1. ...
```

## 不通过验收意见（acceptance_feedback.md）

```markdown
# 不通过验收意见

## 基本信息
- 需求ID：...
- 验收时间：...

## 不通过项清单
1. 不通过项：...
   - 证据：...
   - 影响：...
   - 整改建议：...

## 重规划要求
- 必须在下一次 alexis-plan 中逐条回应并映射到 Task 调整。
```

## 执行计划评审报告（alexis-review）

```markdown
# 执行计划评审报告

## 评审对象
- 需求ID：...
- 需求名称：...

## 总体结论
- 结论：可执行/有条件可执行/不可执行
- 主要原因：...

## 问题清单（按严重度排序）
### P0: <问题标题>
- 定位：Task-XX
- 发现：...
- 影响：...
- 修复建议：...

## 可优化方案
1. ...

## 建议后的 Task 调整
- 需要新增：...
- 需要删除/合并：...
- 需要重排：...

## 问题闭环对照（复评必填）
- 已解决：...
- 部分解决：...
- 未解决：...
- 新增问题：...

## 验收整改覆盖对照（有验收失败时必填）
- 已覆盖的验收意见：...
- 未覆盖的验收意见：...
- 覆盖不足风险：...
```
