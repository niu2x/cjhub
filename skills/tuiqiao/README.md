# 📋 tuiqiao - 需求管理神器

> 把复杂任务变成可追溯、可委派、可跨会话继续的执行闭环 ✨

---

## 💫 为什么需要它？

你是否遇到过这些痛点：

- ❌ 需求讨论了半天，最后还是不知道要做什么
- ❌ 计划写了一堆，执行时发现漏洞百出
- ❌ 任务做到一半，换个会话就忘了进度
- ❌ 验收时才发现偏离了原始目标

**tuiqiao 来拯救你！** 🦸

---

## 🎯 它能做什么？

一条龙搞定需求管理全流程：

| 阶段 | 指令 | 作用 |
|------|------|------|
| 🆕 新建 | `tuiqiao-new` | 澄清需求，直到可执行 |
| 📝 规划 | `tuiqiao-plan` | 起草执行方案 |
| 🔍 评审 | `tuiqiao-review` | 挑剔检查计划 |
| 🔄 迭代 | `tuiqiao-auto-plan` | 自动迭代直到通过 |
| ⚡ 执行 | `tuiqiao-apply` | 串行执行所有任务 |
| ✅ 验收 | `tuiqiao-accept` | 最终确认达标 |

---

## ⚙️ 核心特性

### 📚 文档落库
所有决策和进度都写入文件，**不依赖对话记忆**

### 🔗 会话独立
换个会话也能继续执行，**不怕中断、不怕遗忘**

### 🎯 局部可执行
每个任务独立完整，**委派出去就能开工**

### 🤝 技能协同
自动发现并调用其他技能，**强强联手效率翻倍**

---

## 🚀 快速开始

**第一步：新建需求**

```
tuiqiao-new 实现用户登录功能
```

**第二步：自动规划评审**

```
tuiqiao-auto-plan REQ-001
```

**第三步：执行并验收**

```
tuiqiao-apply REQ-001
tuiqiao-accept REQ-001
```

3. **任务要原子** - 每个任务保持独立可执行
4. **验收要严格** - 不达标就回炉重造
